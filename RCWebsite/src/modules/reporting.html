Reporting Module

Create and manage dynamic Custom Reports

The RC Reporting Module lets you compose easy to read and fully customizable reports in an efficient way. The key feature lies in its automation and templating capabilities.

Unlike common reporting mechanisms RC Reporting works with templates. You simply create and store a template using the RC Template Editor. The Reporting module then manually or automatically outputs reports when data is available. You use variables in the template to determine what data will be loaded and how it will be presented in the report.

There is a variety of usages of the templates and reports. They can be displayed as real-time dashboards, viewed statically or exported to be used in other typical office applications.

Templating

Create Reusuable Templates

The first step in creating a report is creating a template. In the screenshot is an example of a template created in the RC Template Editor.

With the editors easy-to-use interface you create a layout and then assign variables to the cells by typing or using the data browser. The reporting module takes care of the rest.

Using above examples template over two sessions, repeated for every lap.

Reporting

Build real-time Dashboards or Reports

After loading a template you decide which concrete data should be asssigend to the variables by selecting filters related to seasons, races, sessions or drivers. The reporting engine populates the templates variables automatically and transforms it to a full report.

There is no restriction in what data can be used to populate the report. Naturally any data stored in RC Platform can be used but if necessary you can also manually insert data.

Reusing templates for e.g. different sessions isn't a problem. You can also use them for dashboards which update in real-time.

Exporting

Use your reports anywhere

Reports or only specific segements of reports can be exported to common document types without any problems. You can use them where you want.
