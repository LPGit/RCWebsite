RC Platform

Concept

In modern motorsports data efficiency is rapidly becoming the most decisive factor for success. The amount of data produced is growing day by day. Therefore engineers need a modern software solution to handle the large amount of data.

To solve the problem as a whole we built a fast, flexible and easy to use platform. It follows a simple but effective process: Import data, process data and evaluate data.

RC Platforms modular design allows the adaption to any environment. It works in small scale just as well as in large scale. You can just use a single module as well as many.

What it is

RC Platform is a server based software optimized for handling and enhancing your data usage. It can be deployed on a standard computer and is made accessible via your Local Network (LAN) at the race track. Back in the workshop or office you can keep it in your LAN or take it online and access it from the Internet.

First off RC Platform helps you store and collect data. It doesn't really matter which data, any form of source data can be stored. Most data can be collected automatically after a simple configuration e.g. timing data provided by the time keeper or vehicle data recorded by the vehicle logger. For other data such as setup or runsheet data we have specialized modules. E.g. The Runsheet Module will enhance your workflow drastically. To keep up with changing environments and new data sources we designed RC in a modular way. If additional or changing data sources come up RC Platform can swiftly adapt by adding new modules. The collected data is stored in a fast and reliable database in a unified data format that makes further processing easy.

Once the data is stored the analysis process kicks off automatically. Data from one source references data from another source if they are naturally linked. For example a lap time from the timing system corresponds to a lap the race engineer made a comment on in his runsheet. Additionally there can be a video from the tv stream for that particular lap. RC Platform automatically detects and links these three data sets and any other data sets that belong to one another.

The third part of the process is the analysis and evaluation of the data across all the data sources. There are a lot of different modules to utilize the data in a new way. CornerTimes helps you analyze your driver more efficiently. Export modules help you output data for different purposes such as excel like reports or Wintax data structures. And if something new is needed, it can always be developed.

Features

Import any type of source data and create cross source reports and analyses.

Any Source

The modular structure allows any numbers of sources to be imported to the centralized data storage.

High Performance

Optimized code allows maximum performance on import and export operations.

Multi-User Friendly

The server based platform allows multiple users to work on the same data without interfering with each others work.

Customizable

Exports, reports and outputs can be preconfigured creating on the fly outputs mere seconds after the data comes in.

Normalized Data

Switching data supplier is a seamless process for RC Platform thanks to the onboard data normalization. For example Darab data is made compatible with Wintax data.

Module Based

Additional sources, additional outputs, additional analyses. The Modular structure provides quick robust solutions for a dynamic environment.

Small and Large

The platform is built for small as well as large teams.
